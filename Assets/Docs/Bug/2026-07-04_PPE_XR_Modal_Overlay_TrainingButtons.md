## BUG-008: 컨트롤러를 HMD 가까이 당기면 표면이 사라졌다 나타남

### 증상

- 좌우 컨트롤러를 얼굴 쪽으로 당길 때 컨트롤러 메시 일부 또는 전체가 반복적으로 사라졌다 나타난다.

### 원인

- Main Camera의 Near Clip Plane이 `0.3m`였다.
- 컨트롤러 메시가 카메라에서 30cm 이내로 들어오면 근거리 클리핑 평면에 의해 절단됐다.

### 조치

- Main Camera Near Clip Plane을 `0.05m`로 변경했다.

### 남은 검증

- [ ] Quest에서 양쪽 컨트롤러 근접 이동 테스트
- [ ] 낮은 Near Clip 값에 따른 Z-fighting 부작용 확인

## BUG-009: 상세 모달은 보이지만 오버레이가 화면 전체를 가리지 않음

### 증상

- 상세 패널만 팝업처럼 보이고 주변 카드와 HUD가 충분히 어두워지지 않는다.
- Canvas Stretch 적용 후에도 전체 XR 시야가 아니라 기존 HUD 영역만 가려진다.

### 원인

- 부모 Canvas가 월드 스페이스 `1200 × 600` 영역이어서 Stretch가 전체 XR 시야를 의미하지 않았다.
- 카드 내부 Canvas들이 Override Sorting과 Sorting Order `100`을 사용해 일반 Image 오버레이보다 나중에 렌더링됐다.

### 조치

- 오버레이를 중앙 기준 `6000 × 4000` 크기로 확장했다.
- 모달 루트에 Override Sorting Canvas를 적용하고 Sorting Order를 `1000`으로 설정했다.
- 런타임에 Canvas와 GraphicRaycaster 누락 여부를 확인해 보완하도록 했다.

### 남은 검증

- [ ] Game View에서 모든 카드/HUD가 오버레이 아래에 표시되는지 확인
- [ ] Quest 양안 시야 가장자리까지 오버레이가 덮이는지 확인
- [ ] 오버레이가 뒤쪽 카드의 Raycast 입력을 차단하는지 확인

## BUG-010: `훈련하기` 선택 후 세부 시나리오 버튼이 나타나지 않음

### 증상

- `훈련하기` 버튼을 눌러도 요청한 두 시나리오 선택 버튼이 새로 나타나지 않는다.
- 기존 버튼 텍스트만 교체한 초기 구현은 상태 전환을 명확히 보여주지 못했다.

### 원인

- 초기 구현은 새 버튼을 만들지 않고 기존 `훈련하기`와 `돌아가기` 버튼을 재사용했다.
- 플레이 중 스크립트가 갱신되면 기존 런타임 인스턴스에는 새 초기화 로직이 적용되지 않을 수 있다.

### 조치

- Awake에서 기존 버튼을 템플릿으로 다음 버튼 두 개를 별도 생성하고 기본 상태에서는 숨긴다.
  - `PPE 불완전 착용 시나리오`
  - `정상 교육 시나리오`
- `훈련하기` 선택 시 기존 버튼을 숨기고 신규 선택 버튼을 활성화한다.
- 각 버튼에 독립 UnityEvent를 추가했다.
- 긴 한글 레이블에 TMP Auto Size를 적용했다.

### 남은 검증

- Play Mode를 완전히 종료한 뒤 다시 시작해야 런타임 생성 버튼이 반영된다.
- [ ] 두 버튼의 표시, 정렬, 한글 레이블 가독성 확인
- [ ] XR Ray hover 및 Trigger click 확인
- [ ] 선택 이후 연결할 교육 씬 또는 상태 전환 정의
